# 浙仪项目新需求流程

### 应用研究院审批流开始

判断当前审批是否是5大审批之一以及是否选择同步后

+ 属于五大审批流之一并且选择同步，通过实例获取数据，进行手机号和功能校验。
+ 属于五大审批流之一并且选择不同步，直接路过此流程即可。
+ 不属于五大审批流，直接跳过。

+ 如果校验不通过，则进行拒绝处理，并提示失败原因。
+ 如果校验通过，则将审批人手机号对应的userId以及抄送人对应的userId，参数JSON串写入到此子实例对应的数据库中



### 应用研究院审批任务流转开始

判断当前审批是否是5大审批之一以及是否选择同步后

+ 属于五大审批流之一并且选择同步，判断当前任务节点审批人是否为卡点，读取数据库中对应子审批号的记录，获取其审批人userIds和表单数据后发动主组织审批。之后在当前记录下写入生成的主组织审批实例号以及当前审批流的审批人userid。
+ 其他情况直接放弃



### 应用研究院审批流结束

判断当前审批是否是5大审批之一以及是否选择同步后

+ 属于五大审批流之一并且选择同步，读取数据库中对应子审批号的记录，读取消息，获取其抄送人userIds后发动主组织审批(无审批人)



### 控股研究院审批流结束

属于五大指定默认审批，读取数据库获取子审批流程以及taskId，指定通过此taskId的任务审批节点。
